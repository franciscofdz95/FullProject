Business Intelligence & Analysis
By default everyone has access to BIA Security. Users and admins can request/approve access, check user details, edit access and edit application details. The search boxes highlighted in red can be used to filter down to any user or app.
To request access a user can click the blue + button next to an app code to generate a popup.
This popup will have some fields automatically generated. The user will need to fill the Request Geo field from a searchable dropdown (Geo Group and Geo Code will generated based on selection) and Reason. These fields cannot be left blank. Hit Request. The app admin will receive an email about the request.
There are several ways for a user to check their own access. First, click Users and then then the Application # on the right side of any row. This will prompt a popup with all app access and levels. Second, click and based on the icons on the left side a user can quick determine if they have access. The user icon with display a check mark if you have current access and the request blue + button will be hidden.
To approve or deny access an admin will click Access in the top right and then use the search to filter down to the specific user or app. Make sure the Pending toggle is selected at the top of the grid. The actions column can be used to deny or approve the request.
Selecting the My Profile option under Users will show detailed user information. At a glance users can also view all of their access and admins will be able to approve, deny and modify access levels and request.
